Parte 3 · HTML · Componente → etiqueta

Cada componente tiene una etiqueta HTML semántica

Las etiquetas HTML semánticas describen el propósito del contenido, no solo su apariencia.

Encabezado
→
<header>
Contenido principal
→
<main>
Sección temática
→
<section>
Artículo independiente
→
<article>
Pie de página
→
<footer>
Usar la etiqueta correcta hace que el sitio sea más accesible y mejor posicionado en buscadores. No es solo "decoración" del código.
Parte 3 · HTML · Sintaxis de una etiqueta

¿Cómo se escribe una etiqueta HTML?

Toda etiqueta sigue el mismo patrón: apertura, contenido y cierre.

<article class="noticia">

  <h3>El satélite Atenea fue todo un éxito</h3>

  <p>La misión ATFEA completó todos sus objetivos...</p>

  <a href="/noticias/atenea">Leer más</a>

</article>
<article> … </article>
Etiqueta de apertura y de cierre. El contenido va en el medio.
class="noticia"
Atributo: agrega información extra a la etiqueta (nombre de clase CSS).
<h3> <p> <a>
Etiquetas anidadas dentro del artículo: título, párrafo, enlace.
Parte 3 · HTML · Estructura básica del documento

La estructura básica de un documento HTML

<!DOCTYPE html>
<html>

  <head>
    <meta charset="utf-8">
    <title>UBA — Universidad de Buenos Aires</title>
  </head>

  <body>

    <header>
      ... logo, menú de navegación ...
    </header>

    <main>
      <section>
        <article> ... noticia ... </article>
        <article> ... noticia ... </article>
      </section>
    </main>

    <footer>
      ... links, redes sociales, logo ...
    </footer>

  </body>
</html>
<!DOCTYPE html>
Indica al navegador que es HTML5.
<html>
Raíz del documento. Lo contiene todo.
<head>
Configuración (no visible en pantalla).
<body>
Todo lo que se ve en el navegador.
<header>
Encabezado. Igual en todas las páginas.
<main>
Contenido principal. Único por página.
<section>
Sección temática dentro del main.
<article>
Pieza de contenido independiente.
<footer>
Pie de página. Igual en todas las páginas.

Resumen

Lo que aprendimos hoy

1
Un sitio web tiene muchas páginas. Algunas partes se repiten (header, footer) y otras son únicas (el contenido principal).
2
Dentro de una página hay una jerarquía: encabezado → contenido → secciones → artículos → elementos.
3
HTML tiene etiquetas semánticas para cada componente: <header>, <main>, <section>, <article>, <footer>.
